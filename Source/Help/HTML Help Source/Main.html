FPGA Addon

This add-on allows the user to run a FPGA bitfile without needing to implement the VeriStand FPGA Template. The only requirement to the FPGA code is an IRQ (address of 30) to control when the FPGA bitfile starts.

It allows the transfer of basic scalar data types:
  • Bool
  • U8
  • I8
  • U16
  • I16
  • U32
  • I32
  • U64
  • I64
  • SGL
  • FXP +16.3
  • FXP +/-16.5
  • FXP +/-16.7
  • FXP +18.3
  • FXP +/-20.5
  • FXP +/-24.5
  • FXP +/-27.5
  • FXP +/-32.8
  • FXP +/-32.16
  • FXP +/-64.32
The integer data types can be converted into a bitpacked Boolean array.

It adds support for reading/writing waveforms with DMA

  • U8
  • I8
  • U16
  • I16
  • U32
  • I32
  • U64
  • I64
  • SGL
  • FXP +/-32.8
  • FXP +/-32.16
  • FXP +/-64.32
The unsigned integer data types can be converted into a bitpacked Boolean array.

It also supports special FPGA cores (included in the addon folder) for:

  • PWM Generation (PWM.Generate.Digital.vi)
  • Sinewave Generation (Sine.Generate.vi)
  • Digital Static Generation (Static.Generate.Digital.vi)
  • Analog PWM Measurement (PWM.Measure.Analog.vi)
  • Digital PWM Measurement (PWM.Measure.Digital.vi)
  • Digital Static Measurement (Static.Measure.Digital.vi)
  • Digital Wheel Speed Sensor Simulation (WSS.Generate.Digital.vi)
  • Quadrature Encoder Measurement (QuadratureEncoder.Measure.Digital.vi)

What do you want to do?

Disable the Device

Right click on the custom device and select Disable.

Add a Bitfile

To add a bitfile, select the file from the Bitfile path. Note: if you change or re-load the bitfile, all settings will be reset.

Set the RIO Address

Enter in the target FPGA address (usually RIO#) in RIO Device Address.
